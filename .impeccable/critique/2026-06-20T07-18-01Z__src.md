## Anti-Patterns Verdict

**LLM 评审**: 三个改动未引入任何 AI 生成反模式。FormSection 是极简的 fieldset+legend 组件，More 菜单和 View 下拉使用标准 React state+ref+useEffect 模式，无过度工程化、无装饰性填充。

**确定性扫描**: 扫描所有修改文件和核心 UI 组件，返回 **0 条发现**。无渐变文字、无侧边条纹、无眉目标签、无编号标记。

## Overall Impression

上一轮 critique 的三个 P1/P2 问题已全部解决。移动端导航从"导航死路"变为教科书级的 4+More 模式，任务创建从"认知过载"变为清晰的四区 fieldset，任务工具栏从"7+ 控件"降为 4 个。整体分数从 32 提升到 35，进入 Good 区间的上半段。

剩余问题以精细化为主：More 弹出的键盘/ARIA 支持、表单内联验证、翻译一致性。这些问题不影响核心工作流但影响产品的专业完成度。

## What's Working

### 1. 移动端导航是正确的模式
4+More 方案解决了 8 项导航在窄屏上的可达性问题。二级路由高亮（More 按钮在二级页面时显示 accentSoft 背景）维持了方向感。路由切换自动关闭弹出层。

### 2. FormSection 使用了正确的语义 HTML
`<fieldset>` + `<legend>` 是分组表单控件的正确元素，比常见的 `<div>` + `<h3>` 更好，为辅助技术提供了免费的无障碍收益。

### 3. View 下拉平衡了降噪与可达性
自动刷新激活指示点（蓝色圆点）和"对话框打开时暂停"警告都是小但高质量的细节。工具栏从 7+ 控件降至 4 个是显著的认知减负。

## Priority Issues

### [P1] More 菜单弹出缺少键盘和 ARIA 支持
**位置**: `AppShell.tsx` More 按钮和弹出面板
"更多"按钮缺少 `aria-expanded`、`aria-haspopup`，弹出面板无 Escape 关闭处理。TasksPage 的 View 下拉正确实现了全部三项。这是导航主元素上的键盘/无障碍缺口。
**修复**: 添加与 View 下拉相同的 keydown 监听器和 ARIA 属性。
**推荐**: `/impeccable harden`

### [P2] 表单无内联字段验证
**位置**: `CreateTaskDialog.tsx`
验证仅在提交时运行，生成底部单条错误消息。新增的 fieldset 结构为分区/分字段错误指示提供了天然锚点。
**修复**: 无效字段用 `border-app-danger` 高亮，错误消息紧邻字段显示。
**推荐**: `/impeccable harden`

### [P3] "Commit" 未翻译
**位置**: `TasksPage.tsx` MoreActionsMenu 中
菜单项文本硬编码为 "Commit"，同组件其他项均使用 `text()` 双语。
**修复**: 包裹为 `text("Commit", "Commit")` 或找到合适的中文术语。
**推荐**: `/impeccable clarify`

### [P3] 登录页左侧面板在大屏是填充物
**位置**: `LoginPage.tsx` [上轮遗留]
品牌图标+标语+heroCopy 占半屏。可功能化或移除。
**推荐**: `/impeccable distill`

### [P3] EmptyState 缺少视觉区分
**位置**: `DataState.tsx` [上轮遗留]
所有空状态（7+ 处）渲染为居中 muted 文字 + 可选按钮，无上下文图标。
**推荐**: `/impeccable delight`

## Persona Red Flags

### Alex（效率型用户）
会欣赏 fieldset 分区——扫描比逐字段阅读快。会想要一个不经过 View 下拉就能切换自动刷新的快捷键。列设置的双击路径（View → 列设置按钮 → 对话框）可接受但有摩擦。

### Sam（无障碍依赖用户）
**关键**: More 菜单弹出不可键盘操作，无 Escape，无 ARIA。屏幕阅读器用户无法知道 More 按钮打开了面板，也无法通过键盘关闭。FormSection legend 的 `text-xs` 在低视力场景下可能偏小。

### Casey（移动端分心用户）
4+More 是明确改善，不再有被裁剪的导航项。More 弹出无动画显得突兀，slide-up 过渡会更自然。View 下拉的文字标签在窄屏占据工具栏空间。

## Minor Observations

1. More 弹出在 DOM 中渲染于底部导航之后，屏幕阅读器可能无法发现——需要焦点管理
2. View 下拉面板缺少 `role` 属性
3. `text-app-muted` 密集使用使表格偏灰
4. FormSection 组件定义在 CreateTaskDialog 内部，如其他表单需要类似分区应提取为共享组件

## Questions to Consider

1. 自动刷新是否应该住在工具栏？它是后台行为而非主操作——工具栏上一个持续旋转/暂停的 RefreshCw 图标可以替代整个 View 下拉。
2. FormSection 值得提取为共享组件吗？Settings 页面和未来多分区表单都会受益。
3. More 弹出应该是 bottom sheet 而非固定面板吗？在移动端，slide-up overlay + drag-to-dismiss 比固定面板更符合平台惯例。
